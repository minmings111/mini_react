왜 Virtual DOM이 필요한가

Patch 대기

실제 DOM이 느릴 수 있는 이유

실제 DOM은 브라우저 렌더링 엔진과 연결되어 있어 수정될 때 레이아웃 계산(Reflow)과 다시 그리기(Repaint)가 뒤따를 수 있습니다.

Virtual DOM이 하는 일

화면의 구조를 JS 객체 트리로 표현해 두고, 변경 전/후 스냅샷을 비교해 어떤 부분만 바꾸면 되는지 계산합니다.

Diff 5가지 핵심 케이스

  • CREATE: 새 노드 생성
  • REMOVE: 기존 노드 제거
  • REPLACE: 노드 종류나 태그 자체 교체
  • TEXT: 텍스트만 변경
  • ATTRS: 속성만 변경

React와 연결해서 보면
